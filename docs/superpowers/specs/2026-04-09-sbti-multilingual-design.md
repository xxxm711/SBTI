# SBTI 多语言版本设计（简体中文 / English / Espanol）

## 1. 背景

当前 SBTI 页面为单语言（中文）实现，文案大量内嵌在 `sbti_main.js`。目标是在不引入构建工具的前提下，新增完整三语能力，并保持现有测试逻辑、结果计算逻辑与视觉风格基本不变。

## 2. 目标与范围

### 2.1 目标

1. 支持三种语言：`zh-CN`、`en`、`es`。
2. 完整翻译覆盖：
   - 首页、测试页、结果页 UI 文案
   - 全部题目与选项（含特殊题）
   - 人格类型 `intro/desc`
   - 维度名称与 L/M/H 解读
   - 作者区、提示语、按钮等辅助文案
3. 默认语言按浏览器自动识别，未匹配时回退简体中文。
4. 语言切换仅在当前会话生效，不写入本地持久化存储。

### 2.2 非目标

1. 不改动题目评分算法与人格匹配规则。
2. 不改动类型图片资源（`TYPE_IMAGES`）与 `NORMAL_TYPES` 结构。
3. 不引入第三方 i18n 库与打包流程。

## 3. 方案选择

采用 **方案 A：单文件内置字典 + `lang` 状态驱动重渲染**。

选择原因：

1. 与当前项目结构最兼容（纯静态文件）。
2. 实施成本低，能快速得到可用三语版本。
3. 逻辑可控，便于在现有函数中逐步替换文案来源。

## 4. 架构设计

### 4.1 新增语言状态

在 `app` 中新增：

- `app.lang`: 当前语言（`zh-CN | en | es`）

初始化流程：

1. 读取 `navigator.language`。
2. 映射规则：
   - `zh*` -> `zh-CN`
   - `es*` -> `es`
   - 其他 -> `en`
3. 若识别失败，最终回退 `zh-CN`。

### 4.2 字典分层

在 `sbti_main.js` 内引入统一字典对象（示意）：

```js
const I18N = {
  ui: {
    'zh-CN': { ... },
    en: { ... },
    es: { ... }
  },
  questions: {
    'zh-CN': [...],
    en: [...],
    es: [...]
  },
  specialQuestions: {
    'zh-CN': [...],
    en: [...],
    es: [...]
  },
  typeLibrary: {
    'zh-CN': { CTRL: {...}, ... },
    en: { CTRL: {...}, ... },
    es: { CTRL: {...}, ... }
  },
  dimensionMeta: {
    'zh-CN': {...},
    en: {...},
    es: {...}
  },
  dimExplanations: {
    'zh-CN': {...},
    en: {...},
    es: {...}
  }
}
```

### 4.3 数据一致性约束

为确保评分逻辑不变：

1. 各语言题目的 `id`、`dim`、`options.value` 必须严格一致。
2. 人格类型 `code` 必须严格一致。
3. `dimensionOrder` 继续作为唯一计算顺序来源。

## 5. 页面与交互设计

### 5.1 语言切换入口

在 `index.html` 顶部新增简洁语言切换器（例如 3 个按钮）：

- 中文
- EN
- ES

要求：

1. 各屏幕可见（intro/test/result）。
2. 切换时不跳页、不刷新。
3. 样式与现有卡片风格一致，移动端可用。

### 5.2 切换行为

切换语言后执行统一 `rerenderCurrentScreen()`：

1. intro：替换标题和按钮文案。
2. test：
   - 已选答案保留
   - 题干与选项语言刷新
   - 进度与提示语同步刷新
3. result：
   - 保持当前计算结果
   - 仅替换文案与标签语言

### 5.3 会话策略

1. 不使用 `localStorage`、`sessionStorage`。
2. 当前语言只保存在内存变量中。
3. 页面刷新后重新执行浏览器语言识别。

## 6. 关键函数调整

### 6.1 新增工具函数

1. `resolveLang(rawLang)`：归一化语言。
2. `getLocaleBundle(lang)`：获取当前语言字典。
3. `getLocalizedQuestions(lang)`：返回常规题。
4. `getLocalizedSpecialQuestions(lang)`：返回特殊题。
5. `rerenderCurrentScreen()`：按当前屏幕刷新。

### 6.2 现有函数改造点

1. `getVisibleQuestions()`：从语言化题库读取。
2. `getQuestionMetaLabel()`：读 `dimensionMeta[lang]`。
3. `renderQuestions()`：题目文案改为当前语言数据。
4. `updateProgress()`：提示文案从 `ui[lang]` 读取。
5. `computeResult()`：保留计算；类型展示文案改为 `typeLibrary[lang]`。
6. `renderDimList()`：维度名称与解读改为当前语言。
7. `renderResult()`：结果区所有固定文案语言化。

## 7. 回退与错误处理

### 7.1 语言回退

1. 不支持语言 -> `zh-CN`。
2. 指定语言缺失键 -> 回退 `zh-CN` 对应键。

### 7.2 缺失键可观测性

开发态输出警告：

```text
[i18n-missing] lang=es key=result.funNoteSpecial
```

## 8. 样式与可用性

1. 在 `sbti_main.css` 增加语言切换器样式。
2. 保证移动端 320px 宽度下不遮挡主按钮。
3. 对西语较长文本保持换行，不破坏容器边界。

## 9. 验证策略

### 9.1 功能验证

1. 首次进入时按浏览器语言展示。
2. 三语可来回切换且不刷新。
3. 测试页切语言后答案不丢失。
4. 特殊分支正确：
   - 饮酒隐藏人格 `DRUNK`
   - 低匹配兜底 `HHHH`

### 9.2 文案完整性验证

1. 三语下题目总数一致。
2. 三语下结果页全部可见文本已翻译。
3. 作者区与提示语无残留中文（除中文版本）。

### 9.3 回归验证

1. 维度分值、匹配度、主类型与中文原版一致（同一答案输入）。
2. 图片展示逻辑不受影响。

## 10. 风险与缓解

1. **风险：** `sbti_main.js` 已很大，新增三语后体积进一步上升。
   - **缓解：** 保持图片数据不重复，文案结构尽量扁平，后续可再拆分语言包。
2. **风险：** 多语言文本差异导致行高/换行抖动。
   - **缓解：** 对关键区域设置合理 `line-height` 与 `min-height`。
3. **风险：** 键名缺失引发空白文案。
   - **缓解：** 键级回退 + 控制台告警。

## 11. 交付物

1. `index.html`：新增语言切换器结构。
2. `sbti_main.css`：新增切换器样式与响应式细节。
3. `sbti_main.js`：引入完整三语字典与渲染改造。
